3. Fix typography scale to professional, readable sizes:
   - '6xl': ['4rem', { lineHeight: '1' }]      // 64px - hero titles
   - '5xl': ['3rem', { lineHeight: '1' }]      // 48px - page titles
   - '4xl': ['2.25rem', { lineHeight: '1.1' }] // 36px - section headers
   - '3xl': ['1.875rem', { lineHeight: '1.2' }] // 30px - subsection headers

<task type="auto">
  <name>Task 2: Restore Tailwind default spacing system</name>
  <files>tailwind.config.js</files>
  <action>
Remove the custom spacing override entirely from tailwind.config.js theme.extend.

Current custom spacing:
```js
spacing: {
  '1': '0.5rem',   // 8px
  '2': '1rem',     // 16px
  // etc - non-standard
},
```

Replace with: Remove the spacing key entirely. Let Tailwind use its default spacing scale:
- Tailwind default: 1 = 0.25rem (4px), 2 = 0.5rem (8px), 4 = 1rem (16px), 8 = 2rem (32px), etc.
- This is industry standard and what all Tailwind docs reference

WHY: Custom spacing breaks muscle memory, makes Tailwind docs useless, and causes "tasti giganti" because p-3 means 24px instead of expected 12px. Standard spacing gives predictable, professional results.

DO NOT: Create new custom spacing values or try to "improve" Tailwind's spacing. The defaults are battle-tested and professional.

KEEP: colors, fontFamily, fontSize, animation, keyframes, boxShadow (these are good customizations)
  </action>
  <verify>
1. Check tailwind.config.js has no spacing key in theme.extend
2. Check colors, fontFamily, fontSize, animations are still present
3. Run: npm run build -- succeeds without errors
  </verify>
  <done>Custom spacing removed, Tailwind defaults active, build passes</done>
</task>
