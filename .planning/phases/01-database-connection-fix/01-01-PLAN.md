**Problem Statement:**
Users report "file not found" error when trying to scan their Kobo device. The error occurs at line 49 in useKoboDevice.js when calling `getFileByPath(dirHandle, '.kobo/KoboReader.sqlite')`.

**Known Patterns:**
- App uses File System Access API via browser-fs-access polyfill
- Path navigation splits on '/' and navigates through directories
- Kobo database is located at `.kobo/KoboReader.sqlite` (hidden directory)
- Error is FileSystemError with code FS_NOT_FOUND

**Tech Stack:**
- File System Access API (Chrome 86+, Edge 86+)
- browser-fs-access 0.35.0 polyfill
- React 18.3.1 with hooks

**Investigation Areas:**
1. File System Access API behavior with hidden directories (`.kobo`)
2. Case sensitivity of path navigation on different OSes
3. Directory structure actually returned by showDirectoryPicker
4. Whether getFileByPath logic handles all edge cases
</context>
